2nd Delivery

Analysis of the user tasks

We used online surveys to gather information. The distinction between female and male users wasn’t that wide. However, we know this doesn’t show the real state of users in Campus Querétaro, where most of ISC students are male. Ages vary between 18 and 22 with the average being 19.5.

The most used platform by the users are blackboard, Schoology and Google Classroom.

Even though the first opinions about previously used platforms were positive, when asked about more specific tools(such as the ability to read and get files/textbooks from past and future courses.

Our response to user feedback

Users want to be able to review past or future content but are in most cases not able to do it, the main purpose of our website is displaying this information and useful links to students, so our idea makes sense.

Based on feedback, students want a space where they can either help or get help with their courses, so we are going forward with our idea of having a student forum where they can post information or doubts.

Prototype design

We used a palette color with various tones of blue, red and black/white. Most of the figures are round-cornered in the pages. We will have buttons to select between semesters with a drop-out menu including the courses. When the user selects a course, the page will have a table with links divided by either modules or subjects(depends on the course).





Analysis of the implementation of the interaction design principles in the designed system components.

  • We strive to be persistent by using our color palette and the same font(Lato)
  • We search for universal usability with an easy to access and to understand webpage
  • We will include a mobile phone version
  • Feedback will be provided in case of error
  • Every button and section leads to the end of an action.
  • Prevents errors inside the webpage logic and if they appear a message of error will be displayed
  • Allows undo with return options/buttons.
  • Gives control to the user to navigate
  • Simple navigation to avoid overloading the memory of user
  • Emotional aspect avoids false emotions from the computer, focuses on studying and making the user feel comfortable
  • Visual aspects and positive emotions include a balanced color palette, quick and easy to understand interface, a polite feedback and a good webpage organization.
  • Virtual agents won’t be included. Instead, the user can contact directly with the developers.
